API & Extensions

Accueil Architecture Configuration Déploiement API & Extensions

Endpoints HTTP

  • GET / – renvoie templates/index.html.
  • POST /command – reçoit un payload JSON {"action": "forward|backward|…", "speed": }. Actuellement un placeholder qui consigne l’ordre.

Évènements Socket.IO

  • connect – crée un flux dédié au client (thread ou coroutine).
  • disconnect – nettoie le flux.
  • gyro – envoyé toutes les GYRO_SAMPLE_RATE s avec la charge JSON : 
    {
  "accel": {"x":…, "y":…, "z":…},
  "gyro":  {"x":…, "y":…, "z":…},
  "temp":  23.5
}
  • command (client‑to‑server) – décrit ci‑dessus.

Extension future

Le projet est prévu pour accueillir :

  • Un système d’authentification JWT pour sécuriser les connexions Socket.IO.
  • Des contrôleurs de moteurs via la bibliothèque gpiozero ou RPi.GPIO.
  • Une API REST supplémentaire (ex. /status, /metrics) exposée avec Flask‑RESTful.
