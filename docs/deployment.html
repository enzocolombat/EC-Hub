Pré‑requis sur le Raspberry Pi

  • Python 3.9 ou supérieur
  • pip installé
  • Accès I²C activé (sudo raspi-config → Interface Options → I2C → enable)
  • Connexion réseau fonctionnelle (Wi‑Fi ou Ethernet)

Installation des dépendances sur le Pi

# Sur le Pi
sudo apt-get update && sudo apt-get install -y python3-venv git
git clone https://github.com//robot-sensor-server.git
cd robot-sensor-server
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

Lancer le serveur

# Option 1 – lancement manuel
source .venv/bin/activate
python -m Robot.server

# Option 2 – service systemd (recommandé)
# Créez /etc/systemd/system/robot-sensor.service avec le contenu suivant :
[Unit]
Description=Robot Sensor Server
After=network.target

[Service]
WorkingDirectory=/home/pi/robot-sensor-server
ExecStart=/home/pi/robot-sensor-server/.venv/bin/python -m Robot.server
Restart=on-failure
User=pi
EnvironmentFile=/home/pi/robot-sensor-server/.env

[Install]
WantedBy=multi-user.target

# Puis activez et démarrez :
sudo systemctl daemon-reload
sudo systemctl enable robot-sensor.service
sudo systemctl start robot-sensor.service

Déploiement depuis Windows (script existant)

Le fichier deploy.bat lit les variables PI_USER, PI_PASS et PI_IP depuis .env puis copie les fichiers via WinSCP.

Points à améliorer :

  • Passer à un script multiplateforme (PowerShell ou Python) ; le batch ne fonctionne que sous Windows.
  • Ne pas stocker le mot de passe en clair dans .env. Utiliser des clés SSH ou un gestionnaire de secrets.
  • Ajouter la création d’un virtualenv et l’installation des dépendances sur le Pi après le transfert.

Docker (optionnel)

# Dockerfile (à placer à la racine)
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 5000
CMD ["python", "-m", "Robot.server"]

Construisez et lancez :

docker build -t robot-sensor .
docker run -d -p 5000:5000 --name robot-sensor robot-sensor
