Vue globale

Le système se compose de trois couches principales :

  1. Capteur matériel : MPU‑6050 connecté via I²C.
  2. Backend Python : Flask + Flask‑Socket.IO.
    • Le module Robot/sensors/gyro.py encapsule le driver mpu6050 et expose get_data().
    • Le serveur (Robot/server.py) crée un SocketIO serveur, gère les connexions WebSocket et diffuse les mesures toutes les GYRO_SAMPLE_RATE secondes.
    • Un endpoint /command est prévu pour recevoir des actions (ex. contrôle moteur).
  3. Frontend web : HTML + CSS + JavaScript.
    • Interface/templates/index.html charge la page et inclut les scripts.
    • Interface/static/robot.js, socket.js, canvas.js gèrent la connexion Socket.IO, la mise à jour du DOM et le rendu du canvas.

Flux de données

Capteur MPU‑6050 → gyro.py (get_data) → server.py (thread _gyro_stream) → Socket.IO « gyro » → client JavaScript → DOM / Canvas
